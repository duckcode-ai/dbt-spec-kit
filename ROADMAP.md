# Roadmap

dbt-spec-kit is focused on controlled autonomy for dbt teams: agents can help implement work, but
specs, plans, tests, and CI remain the trust boundary.

## Near term

- Polish onboarding around the upstream dbt Labs jaffle-shop project.
- Add more real-world examples from anonymized dbt projects.
- Improve manifest-aware validators for semantic models, metrics, saved queries, and exposures.
- Add Redshift warehouse preset.
- Add docs checks so README and onboarding links do not drift.

## Later

- Add optional policy configuration for warning vs. error severity.
- Add more warehouse-specific policy packs.
- Add PR-comment integration examples for GitHub Actions.
- Add migration guidance for teams moving from ad hoc agent prompts to spec-driven delivery.

## Not planned

- Hosted service requirement.
- Telemetry by default.
- A replacement for dbt, dbt Cloud, or dbt Labs agent skills.
